Semântica na html

Semântica: significado dos vocábulos, por oposição à sua forma.

A relação entre semântica e a html, é que a semântica da mais valor ao significado da palavra do que sua forma.
Da mesma maneira a html5, que possue tags com significado, deixando as tags da html4 que só mudavam o formato obsoletas. Atualmente, a html foca unicamente no conteúdo semântico, deixando a forma do texto unicamente para as CSS.
Logo algumas tags que só realizam formatações visuais devem ser substituidas, como:

<b>(bold) deve ser substituida por <strong>.
<i>(italico) deve ser substituida por <em>.

Para marcar texto use a tag <mark>.

Obiviamente, tem algumas exceções como a tag <small> que continua sendo usada, diferente da sua semelhante <big>.

A tag <del> significa textos que devem ser desconsiderados.

Já a tag <ins> significa textos que devem ser lidos com atenção.
Ela substitui a tag <u> que apenas sublinhava os textos

A tags <sub> e <sup>, para textos subscrito e sobreescirto. 

            As tags <code>, para indicar um código 
            e <pre> para manter a formatação do documento.

Para fazer citações no meio de paragrafos use a tag <q>
Já se quiser um paragrafo inteiro use a tag <blockquote>, usando o parametro cite para linkar o texto original:

Não tenho cor, nem madrinha
Nasci no mundo, sou sozinha
Não tenho pressa, não tenho plano, não tenho dono

Na HTML5 também se usa a tag <abbr> para marcar abreviações, com o "title" para mostrar seu significado.

E a tag mais importante de todas, a <bdo>, com o parametro "dir", par sinalizar par que direção o texto será invertido:
Do caralho essa aqui tá.
